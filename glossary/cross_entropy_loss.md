# Cross-Entropy Loss (ฟังก์ชันความสูญเสียเอนโทรปีข้าม)

[← กลับสู่เนื้อหาบทเรียน](../lectures/project1_llm_foundations.md)

**Cross-Entropy Loss** หรือ Log Loss เป็นฟังก์ชันความสูญเสีย (Loss Function) ยอดนิยมที่สุดในการฝึกสอนโมเดล Machine Learning สำหรับปัญหาการจำแนกประเภท (Classification Problems) และใช้เป็นแกนหลักในการฝึกฝนโมเดลภาษาขนาดใหญ่ (LLMs) เพื่อทำนายโทเคนถัดไป

## คำอธิบายเชิงลึก (In-depth Explanation)
ในทฤษฎีข้อมูล (Information Theory) เอนโทรปีข้าม (Cross-Entropy) วัดว่าต้องใช้ข้อมูล (บิต) เท่าใดในการระบุเหตุการณ์จากการกระจายตัวความน่าจะเป็น $P$ (ค่าจริง) เมื่อเราใช้การกระจายตัว $Q$ (ค่าที่โมเดลคาดคะเน) แทน

สมการทางคณิตศาสตร์สำหรับคลาสจำนวน $C$ คลาส คือ:
$$\mathcal{L}_{CE} = - \sum_{i=1}^C y_i \log(\hat{y}_i)$$

โดยที่:
*   $y_i$ คือค่าเฉลยความจริง (Ground truth) ซึ่งโดยปกติจะเป็นแบบ One-hot encoding (คือเป็น 1 ที่คลาสที่ถูกต้อง และ 0 สำหรับคลาสอื่นทั้งหมด)
*   $\hat{y}_i$ คือความน่าจะเป็นที่โมเดลพยากรณ์ออกมาสำหรับคลาสนั้น (ปกติได้มาจากฟังก์ชัน Softmax)

เนื่องจาก $y_i$ มีค่าเป็น 0 สำหรับคลาสที่ผิดทั้งหมด สมการด้านบนจึงมักหดตัวลงเหลือเพียง:
$$\mathcal{L}_{CE} = - \log(\hat{y}_{\text{correct\_class}})$$

นี่คือเหตุผลที่มันถูกเรียกว่า **Negative Log-Likelihood (NLL)** นั่นคือ โมเดลจะถูกทำโทษ (Loss สูง) ถ้าระบุความน่าจะเป็นให้กับคำตอบที่ถูกไว้น้อย และจะได้รางวัล (Loss ต่ำเข้าใกล้ศูนย์) ถ้าระบุความน่าจะเป็นให้คำตอบที่ถูกไว้สูงใกล้เคียง 1.0

## ตัวอย่างการนำไปใช้ (Practical Example)
ในการเทรน LLM ให้สร้างประโยค "แมว กิน ปลา"
สมมติว่าโมเดลอ่านถึง "แมว กิน" แล้วต้องพยากรณ์คำถัดไปคือ "ปลา" (คลาสเป้าหมาย)
*   **กรณีโมเดลโง่**: ทำนายว่ามีโอกาสเป็น "ปลา" แค่ 10% (0.1) -> Loss $= -\log(0.1) \approx 2.30$
*   **กรณีโมเดลเก่ง**: ทำนายว่ามีโอกาสเป็น "ปลา" ถึง 90% (0.9) -> Loss $= -\log(0.9) \approx 0.10$

ตัวปรับค่าน้ำหนัก (Optimizer) จะพยายามจูนพารามิเตอร์เพื่อดึงให้ค่า Loss ตรงนี้ต่ำที่สุดเท่าที่จะทำได้ในชุดข้อมูลฝึกสอน
