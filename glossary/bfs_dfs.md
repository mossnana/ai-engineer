# Breadth-First Search (BFS) และ Depth-First Search (DFS)

[← กลับสู่เนื้อหาบทเรียน](../lectures/project4_deep_research_reasoning.md)

**BFS (การค้นหาตามแนวกว้าง)** และ **DFS (การค้นหาตามแนวลึก)**
เป็นอัลกอริทึมพื้นฐานที่สำคัญที่สุดในวิทยาการคอมพิวเตอร์ สำหรับใช้ท่องไป (Traverse) หรือค้นหาข้อมูล
(Search) ภายในโครงสร้างข้อมูลประเภทต้นไม้ (Tree) และกราฟ (Graph)

## คำอธิบายเชิงลึก (In-depth Explanation)

ในการออกแบบระบบ AI ที่ต้องค้นหาคำตอบหรือสืบหาเส้นทาง (เช่น Agent
ที่ต้องหาข้อมูลจากอินเทอร์เน็ตเพื่อทำรายงาน) โครงสร้างการค้นหาจะถูกมองเป็นต้นไม้ของการตัดสินใจ
การเลือกว่าจะใช้ BFS หรือ DFS จะส่งผลต่อพฤติกรรมและความเร็วของ Agent อย่างมาก

### 1. Breadth-First Search (BFS)

- **หลักการ**: สำรวจโหนดที่อยู่ระดับตื้นที่สุด (ใกล้จุดเริ่มต้นที่สุด) ให้ครบทุกโหนดในชั้นนั้นก่อน แล้วค่อยขยับลงไปสำรวจในชั้นถัดไปที่ลึกขึ้น (สำรวจแนวนอน)
- **โครงสร้างข้อมูลที่ใช้**: คิว (Queue - First In First Out)
- **ข้อดี**: การันตีว่าจะเจอเส้นทางที่สั้นที่สุด (Shortest Path) เสมอ (ในกรณีที่ขอบมีน้ำหนักเท่ากัน)
- **ข้อเสีย**: กินหน่วยความจำมหาศาล (Exponential memory) เพราะต้องเก็บข้อมูลของโหนดในชั้นเดียวกันทั้งหมดไว้ในหน่วยความจำ
- **ตัวอย่างใน AI**: ระบบ Web Crawler ที่ตั้งค่าให้อ่านหัวข้อข่าวหน้าแรกทั้งหมดก่อน แล้วค่อยเจาะเข้าไปอ่านเนื้อหาข้างในทีละข่าว

> 💡 **เปรียบเทียบให้เห็นภาพ BFS (Analogy):** สมมติว่าคุณทำกุญแจหายในตึก 3 ชั้น การหาแบบ BFS คือคุณต้องเดินหาให้ทั่วชั้น 1 ทุกซอกทุกมุมจนครบก่อน ถ้าไม่เจอค่อยเดินขึ้นไปหาชั้น 2 ให้ครบทุกห้อง แล้วค่อยไปชั้น 3 วิธีนี้มั่นใจได้ว่าคุณจะไม่พลาดอะไรใกล้ๆ ตัวไป แต่ข้อเสียคือคุณต้องจำให้ได้ว่าห้องไหนชั้นนี้หาไปแล้วบ้าง

### 2. Depth-First Search (DFS)

- **หลักการ**: สำรวจเจาะลึกลงไปตามกิ่งใดกิ่งหนึ่งให้ถึงจุดลึกสุดก่อน หากไปต่อไม่ได้แล้วจึงถอยกลับ (Backtrack) ขึ้นมาหนึ่งก้าวเพื่อไปสำรวจกิ่งอื่นที่ยังไม่ได้ไป (สำรวจแนวตั้ง)
- **โครงสร้างข้อมูลที่ใช้**: สแตก (Stack - Last In First Out) หรือการเขียนฟังก์ชันแบบเรียกซ้ำ (Recursion)
- **ข้อดี**: กินหน่วยความจำน้อยมาก (แปรผันตามความลึก) และเหมาะกับการแก้ปัญหาประเภทหาทางออกจากเขาวงกต
- **ข้อเสีย**: อาจไปติดแหงกอยู่ในกิ่งที่ลึกมากๆ โดยไม่รู้จบ (Infinite loop) หรืออาจเจอคำตอบที่อยู่ในกิ่งลึกซึ่งไม่ใช่ทางออกที่ดีที่สุด
- **ตัวอย่างใน AI**: ระบบ AlphaGo เมื่อจำลองเดินหมากตาล่วงหน้า (Monte Carlo Tree Search) จะทำการ Rollout จำลองดำดิ่งลงไปจนสุดจบเกม เพื่อดูผลลัพธ์ว่าชนะหรือแพ้ แล้วค่อยถอยกลับมาอัปเดตคะแนน

> 💡 **เปรียบเทียบให้เห็นภาพ DFS (Analogy):** การหาของแบบ DFS คือคุณเดินจากห้องนั่งเล่นชั้น 1 ทะลุเข้าห้องนอน เปิดตู้เสื้อผ้า มุดเข้าไปในลิ้นชักจนสุดทาง ถ้าไม่เจอก็ถอยออกมาแค่หน้าตู้แล้วหาลิ้นชักถัดไป คือขุดเจาะลงไปให้สุดทางใดทางหนึ่งก่อนเลย ข้อดีคือถ้าของบังเอิญอยู่ลึกคุณจะเจอเร็วมากและไม่ต้องจำเยอะ แต่ข้อเสียคือคุณอาจจะเสียเวลาเป็นวันๆ ขุดหาในห้องนอน ทั้งๆ ที่กุญแจวางอยู่บนโต๊ะกินข้าวชั้น 1
