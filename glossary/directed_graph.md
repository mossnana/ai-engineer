# Directed Graph (กราฟระบุทิศทาง)

[← กลับสู่เนื้อหาบทเรียน](../lectures/project3_agentic_workflows_tools.md)

**Directed Graph (Digraph)** หรือ กราฟระบุทิศทาง เป็นโครงสร้างข้อมูลในทฤษฎีกราฟและคณิตศาสตร์แบบไม่ต่อเนื่อง (Discrete Mathematics) ซึ่งประกอบด้วยจุดยอด (Vertices/Nodes) และเส้นเชื่อม (Edges) โดยที่เส้นเชื่อมแต่ละเส้นจะมี "ทิศทาง" กำกับไว้อย่างชัดเจน (คล้ายกับถนนแบบวันเวย์)

## คำอธิบายเชิงลึก (In-depth Explanation)
ในทางคณิตศาสตร์ กราฟระบุทิศทาง $G$ นิยามโดยเซตคู่ลำดับ $G = (V, E)$:
*   $V$ คือเซตของจุดยอด (Nodes)
*   $E$ คือเซตของเส้นเชื่อม โดยที่แต่ละเส้นเชื่อมเป็นคู่ลำดับ $(u, v)$ ซึ่งหมายความว่ามีเส้นทางชี้จากโหนด $u$ ไปหาโหนด $v$ เท่านั้น ไม่สามารถเดินทางย้อนกลับจาก $v$ ไป $u$ ได้โดยอัตโนมัติ (เว้นแต่จะมีเส้นเชื่อมอีกเส้น $(v, u)$ กำกับไว้ต่างหาก)

เมื่อเทียบกับ Undirected Graph (กราฟไม่มีทิศทาง) ที่เปรียบเสมือนเครือข่ายเพื่อนใน Facebook (A เป็นเพื่อนกับ B แสดงว่า B ก็เป็นเพื่อนกับ A) Directed Graph จะเปรียบเสมือนเครือข่ายผู้ติดตามใน Twitter/X (A ตาม B แต่ B ไม่จำเป็นต้องตาม A)

**Directed Acyclic Graph (DAG)** คือรูปแบบเฉพาะของ Directed Graph ที่จะไม่มีการวนลูป (Cycle) กลับมาที่จุดเดิมได้เลย โครงสร้าง DAG นิยมใช้มากในการสร้าง Workflow, แผนภูมิกระบวนการงาน (เช่น Apache Airflow), การเรียงลำดับความสำคัญของงาน (Topological Sort), หรือ State Machine ของ AI Agent

## ตัวอย่างการนำไปใช้ (Practical Example)
*   **LangGraph / AI Agent State Machines**: โครงสร้างการทำงานของ Multi-Agent มักใช้ Directed Graph เป็นตัวกำหนดทิศทางข้อมูล เช่น ข้อมูลวิ่งจาก (โหนดรับคำสั่ง) $\rightarrow$ (โหนดนักวิจัย) $\rightarrow$ (โหนดสรุปผล) แต่ห้ามวิ่งย้อนกลับ
*   **หน้าเว็บเพจ**: เครือข่ายหน้าเว็บทั้งหมดบนโลกอินเทอร์เน็ต (WWW) คือตัวอย่างของ Directed Graph ขนาดมหึมา หน้าเว็บ A อาจมี Hyperlink โยงไปหน้าเว็บ B (ขอบระบุทิศทาง) แต่หน้าเว็บ B อาจไม่ได้โยงกลับมาที่ A
