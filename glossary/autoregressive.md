# Autoregressive (ออโตรีเกรสซีฟ)

[← กลับสู่เนื้อหาบทเรียน](../lectures/project1_llm_foundations.md)

**Autoregressive** (มักย่อว่า AR) เป็นคำที่มาจากวิชาสถิติ (Time Series Analysis)
หมายถึงแบบจำลองที่ทำนายค่าในอนาคตโดยอาศัยค่าของตัวมันเองในอดีตมาเป็นตัวแปรต้น (Auto = ตัวเอง,
Regress = การถดถอย/การวิเคราะห์หาความสัมพันธ์)

ในบริบทของ Machine Learning และ LLM (เช่น ตระกูล GPT) โมเดลแบบ Autoregressive
คือโมเดลที่สร้างข้อมูลออกมาทีละหนึ่งหน่วย (เช่น ทีละหนึ่งโทเคน) ตามลำดับเวลา
และนำข้อมูลที่เพิ่งสร้างเสร็จนั้นป้อนกลับเข้าไปเป็นส่วนหนึ่งของอินพุตเพื่อใช้ทำนายข้อมูลในลำดับถัดไป

> 💡 **เปรียบเทียบให้เห็นภาพ (Analogy):** ลองนึกถึงการเล่นเกม "แต่งนิทานคนละคำ" กับเพื่อน
> คุณจะพูดคำต่อไปได้ คุณต้องรอฟังคำที่เพื่อนเพิ่งพูดจบเสียก่อน แล้วเอาคำของเพื่อนมารวมกับเนื้อเรื่องเดิม เพื่อแต่งคำของคุณต่อ
> ทันทีที่คุณพูดจบ คำของคุณก็จะกลายเป็นอดีตให้เพื่อนคนต่อไปใช้แต่งเรื่องต่อ วนไปเรื่อยๆ จนกว่านิทานจะจบ

## คำอธิบายเชิงลึก (In-depth Explanation)

สมการของโมเดลภาษาแบบ Autoregressive สามารถเขียนได้ดังนี้:
$$y_t = f_\theta(y_1, y_2, \dots, y_{t-1})$$

ที่เวลา $t$ โมเดลจะใช้ข้อมูลทั้งหมดตั้งแต่ $y_1$ ถึง $y_{t-1}$ ในการพยากรณ์ $y_t$ ทันทีที่ได้
$y_t$ มา $y_t$ ก็จะถูกรวมเข้าไปในเซ็ตข้อมูลเพื่อนำไปพยากรณ์ $y_{t+1}$ ต่อไป
_(เหมือนการอ่านเรื่องทั้งหมดที่แต่งมาก่อนหน้า แล้วเดาว่าคำต่อไปควรจะเป็นอะไร)_

ข้อดีของสถาปัตยกรรมนี้คือสร้างข้อความได้ลื่นไหลและเป็นธรรมชาติ แต่ข้อเสียที่สำคัญคือ **คำนวณช้า (High
Latency) ระหว่างการ Inference** เพราะการสร้างประโยคความยาว $N$ คำ ต้องรันโมเดล
(Forward pass) จำนวน $N$ รอบเรียงลำดับกันไป ไม่สามารถทำแบบขนาน (Parallelize)
ในขั้นตอนการสร้างคำตอบได้เลย
_(เหมือนเกมแต่งนิทานที่ต้องรอคนก่อนหน้าพูดจบ จะข้ามคิวพูดพร้อมกันหลายๆ คนไม่ได้ ทำให้เสียเวลา)_

## ตัวอย่างการนำไปใช้ (Practical Example)

เวลาใช้งาน ChatGPT เมื่อคุณพิมพ์คำสั่ง โมเดลจะไม่ได้ตอบออกมาทั้งประโยคในเสี้ยววินาที
แต่คุณจะเห็นข้อความค่อยๆ "พิมพ์" ออกมาทีละคำ นั่นเป็นเพราะโมเดลกำลังทำงานแบบ Autoregressive:
สร้างคำแรก -> เอาคำแรกไปต่อท้ายคำสั่งเดิม -> คำนวณคำที่สอง -> เอาคำที่สองไปต่อท้าย ->
คำนวณคำที่สาม วนลูปไปเรื่อยๆ จนกว่าจะเจอ Stop Token
