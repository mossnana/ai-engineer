# CI/CD (Continuous Integration และ Continuous Deployment)

[← กลับสู่เนื้อหาบทเรียน](../lectures/project6_capstone_guide.md)

**CI/CD** เป็นปรัชญาและชุดกระบวนการปฏิบัติ (Practices) สำคัญในวงการวิศวกรรมซอฟต์แวร์สมัยใหม่ (DevOps) โดยมีจุดประสงค์หลักเพื่อลดช่องว่างของเวลาระหว่างการเขียนโค้ดของนักพัฒนา ไปจนถึงวันที่โค้ดนั้นถูกติดตั้งและให้บริการจริงแก่ลูกค้า (Time-to-market) ทำให้ซอฟต์แวร์สามารถปรับปรุงอัปเดตตัวเองได้อย่างต่อเนื่องและอัตโนมัติ

## คำอธิบายเชิงลึก (In-depth Explanation)

กระบวนการนี้แบ่งออกเป็น 2 ส่วนหลัก:

### 1. Continuous Integration (CI) - การบูรณาการอย่างต่อเนื่อง
*   คือกฎที่ให้นักพัฒนาทุกคนต้องนำโค้ดที่เขียนเสร็จแล้ว ไปรวมกับโค้ดหลัก (Merge to Main Branch) อย่างสม่ำเสมอ เช่น วันละหลายๆ ครั้ง (แทนที่จะทำฟีเจอร์ใหญ่ 3 เดือนแล้วค่อยเอามารวมกัน ซึ่งจะเจอบั๊กมหาศาล)
*   **หัวใจสำคัญ**: ทุกครั้งที่มีโค้ดใหม่ถูกส่งเข้ามา (Push) ระบบเซิร์ฟเวอร์ (เช่น GitHub Actions, GitLab CI, Jenkins) จะทำงานอัตโนมัติเพื่อ **"Build โค้ด"** และ **"Run Unit Tests ทั้งหมด"** 
*   ถ้าระบบทดสอบผ่าน ถือว่าโค้ดนั้นปลอดภัย ถ้าไม่ผ่าน ระบบจะ Reject โค้ดนั้นทิ้งทันทีพร้อมแจ้งเตือน

### 2. Continuous Delivery / Continuous Deployment (CD) - การส่งมอบ/ติดตั้งอย่างต่อเนื่อง
*   เมื่อโค้ดผ่านขั้นตอน CI มาแล้ว ขั้นตอน CD จะเข้ามารับช่วงต่อ โดยนำโค้ดที่แพ็กเรียบร้อย (เช่น Docker Image) ไปติดตั้ง (Deploy) ลงบนเซิร์ฟเวอร์ของจริง (Production) หรือเซิร์ฟเวอร์ทดสอบ (Staging) 
*   กระบวนการนี้อาจทำโดยอัตโนมัติ 100% เลย (Continuous Deployment) หรืออาจให้คนกดปุ่มอนุมัติ 1 ครั้ง (Continuous Delivery) ก็ได้

## ตัวอย่างการนำไปใช้ (Practical Example)
การพัฒนา AI Application ใน Production:
เมื่อคุณเทรนโมเดลตัวใหม่เสร็จ และแก้โค้ดฝั่ง Backend (FastAPI) เพื่อเรียกใช้โมเดลตัวใหม่ 
*   **CI**: ทันทีที่คุณกด `git push` ระบบ GitHub Actions จะดึงโค้ดคุณไปรันสคริปต์ทดสอบ (เช่น ลองยิง Prompt ปลอมๆ ดูว่าโมเดลตอบถูกไหม, ดูว่าเว็บไม่ล่มใช่ไหม)
*   **CD**: ถ้าทุกอย่างเป็นสีเขียว (ผ่าน) ระบบจะสร้าง Docker Image ก้อนใหม่ อัปโหลดขึ้น AWS/GCP แล้วสั่งลบเซิร์ฟเวอร์ตัวเก่า สตาร์ทเซิร์ฟเวอร์ตัวใหม่ให้ผู้ใช้งานทั่วโลกได้ใช้โมเดลตัวใหม่ของคุณทันที ภายในเวลา 5 นาทีหลังกด Push
