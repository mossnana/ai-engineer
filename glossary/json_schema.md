# JSON Schema (เจซัน สคีมา)

[← กลับสู่เนื้อหาบทเรียน](../lectures/project3_agentic_workflows_tools.md)

**JSON Schema** เป็นมาตรฐานสำหรับการอธิบายและตรวจสอบ (Validate) โครงสร้างข้อมูลที่อยู่ในรูปแบบเอกสาร JSON (JavaScript Object Notation)

ทำหน้าที่คล้ายกับแม่พิมพ์ (Blueprint) หรือพิมพ์เขียวที่คอยตั้งกฎเกณฑ์ว่า ข้อมูล JSON ที่รับมานั้น "ถูกต้อง" ตามรูปแบบที่เราต้องการหรือไม่ (เช่น ต้องมีฟิลด์อะไรบ้าง, ฟิลด์ไหนบังคับ/ไม่บังคับ, ประเภทข้อมูลต้องเป็น String หรือ Number)

## คำอธิบายเชิงลึก (In-depth Explanation)

ในวงการ AI และ Tool Calling / Function Calling ของ LLMs, JSON Schema มีความสำคัญระดับคอขาดบาดตาย เพราะ LLM เป็นเพียงตัวสร้างข้อความ (Text Generator) ไม่ใช่โปรแกรมคอมพิวเตอร์ที่เคร่งครัดเรื่อง Type หากเราปล่อยให้มันเรียกใช้ฟังก์ชันอิสระ มันอาจส่งข้อความสตริงมาให้พารามิเตอร์ที่ต้องการตัวเลข ทำให้ระบบแครชทันที

เราจึงต้องส่ง JSON Schema ไปพร้อมกับคำอธิบายเครื่องมือ (Tool Description) เพื่อบังคับให้ LLM รู้ว่ามันจะต้องส่งข้อมูลกลับมาในรูปแบบไหน

ตัวอย่าง JSON Schema เบื้องต้น:

```json
{
  "type": "object",
  "properties": {
    "name": { "type": "string" },
    "age": { "type": "integer", "minimum": 0 },
    "email": { "type": "string", "format": "email" }
  },
  "required": ["name", "age"]
}
```

จากสคีมาด้านบน ระบบรับประกันได้ว่าข้อมูลที่จะผ่านเข้าไปในโปรแกรมได้ **ต้อง**มีฟิลด์ name และ age เสมอ โดยที่ age ต้องเป็นตัวเลขจำนวนเต็มที่ไม่ติดลบ และอีเมล (ถ้ามี) ต้องอยู่ในรูปแบบอีเมลที่ถูกต้อง

## ตัวอย่างการนำไปใช้ (Practical Example)

การนิยาม Tool สำหรับ LLM เพื่อพยากรณ์อากาศ:
เมื่อคุณสร้าง Tool `get_weather` ให้ ChatGPT ใช้ คุณจะต้องแนบ JSON Schema กำหนดว่ารับพารามิเตอร์ 1 ตัวคือ `location` (ประเภท String) โมเดลจะอ่าน Schema นี้แล้วสร้างคำสั่งเรียก Tool ออกมาเป็น JSON ที่ถูกต้อง เช่น `{"location": "Bangkok"}` เพื่อให้โปรแกรมเมอร์ดึงไปรันต่อได้อย่างปลอดภัย
