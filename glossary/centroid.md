# Centroid (เซนทรอยด์)

[← กลับสู่เนื้อหาบทเรียน](../lectures/project2_rag_prompt_engineering.md)

**Centroid** (เซนทรอยด์) ในทางเรขาคณิตและวิทยาศาสตร์ข้อมูล หมายถึง **"จุดศูนย์กลางมวล"**
หรือตำแหน่งค่าเฉลี่ยตรงกลางของรูปทรงเรขาคณิตหรือของกลุ่มข้อมูลในปริภูมิ $N$ มิติ

> 💡 **เปรียบเทียบให้เห็นภาพ (Analogy):** ลองนึกภาพคุณกับเพื่อนอีก 4 คน บ้านอยู่คนละทิศคนละทางในกรุงเทพฯ แล้วพวกคุณอยากจะนัดเจอกันที่ร้านอาหารสักร้าน โดยมีเงื่อนไขว่า "ทุกคนต้องเดินทางมาร้านนี้ในระยะทางที่เฉลี่ยแล้วแฟร์ที่สุด" จุดพิกัดของร้านอาหารร้านนั้นแหละครับที่เรียกว่า **Centroid** (จุดศูนย์กลางของกลุ่มเพื่อนคุณ)

## คำอธิบายเชิงลึก (In-depth Explanation)

ในทาง Machine Learning โดยเฉพาะในการจัดกลุ่มข้อมูล (Clustering algorithms) อย่างเช่น
[K-Means](k_means_clustering.md) Centroid คือตัวแทนพิกัดใจกลางของข้อมูลกลุ่มนั้นๆ

การคำนวณตำแหน่งพิกัดของ Centroid $\mu$ สำหรับกลุ่มข้อมูล $S$ ที่มีสมาชิก
$x_1, x_2, \dots, x_n$ ทำได้โดยการหาค่าเฉลี่ยทางคณิตศาสตร์ของพิกัดในทุกๆ
มิติของจุดสมาชิกในกลุ่มนั้น: $$\mu = \frac{1}{|S|} \sum_{x \in S} x$$
_(เหมือนเอาละติจูดและลองจิจูดของบ้านเพื่อนทุกคนมาบวกกันแล้วหารด้วยจำนวนคน เพื่อหาพิกัดกึ่งกลาง)_

## ตัวอย่างการนำไปใช้ (Practical Example)

- **Vector Database & IVF-PQ**: ในการบีบอัดฐานข้อมูลเวกเตอร์แบบ PQ (Product
  Quantization) แทนที่ระบบจะเก็บเวกเตอร์ทศนิยมตัวจริงของเอกสารแบบเต็มๆ
  ระบบจะจัดกลุ่มเวกเตอร์ออกเป็นหลายๆ กลุ่มย่อย แล้วประเมินหา **Centroid** ของแต่ละกลุ่มย่อย
  จากนั้นจะเก็บเฉพาะ **หมายเลขไอดีของ Centroid (Byte ID)** ที่ใกล้กับเอกสารนั้นที่สุดแทน
  ทำให้บีบอัดพื้นที่จัดเก็บได้เล็กลงหลายสิบเท่า _(เหมือนจำแค่ว่า "นัดแถวสยาม" แทนที่จะจำละติจูดลองจิจูดเป๊ะๆ ของร้านอาหาร)_
- ตัวอย่าง 2 มิติ: ถ้าคุณมีกลุ่มจุดพิกัดในกราฟ 3 จุดคือ $(1, 1), (3, 1), (2, 4)$
  - พิกัดแกน x ของเซนทรอยด์ = $(1 + 3 + 2) / 3 = 2$
  - พิกัดแกน y ของเซนทรอยด์ = $(1 + 1 + 4) / 3 = 2$
  - ดังนั้น Centroid ของกลุ่มนี้อยู่ที่พิกัด $(2, 2)$
