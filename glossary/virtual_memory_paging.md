# Virtual Memory Paging (ระบบหน้าหน่วยความจำเสมือน)

[← กลับสู่เนื้อหาบทเรียน](../lectures/project4_deep_research_reasoning.md)

**Virtual Memory Paging** เป็นเทคโนโลยีระดับระบบปฏิบัติการ (OS) สำหรับการบริหารจัดการหน่วยความจำ (RAM) โดยมีแนวคิดหลักคือการ "หลอก" ให้โปรแกรมหรือแอปพลิเคชันคิดว่ามันมีหน่วยความจำให้ใช้แบบต่อเนื่องและมีขนาดใหญ่กว่าที่ฮาร์ดแวร์มีอยู่จริง

## คำอธิบายเชิงลึก (In-depth Explanation)

ในยุคคอมพิวเตอร์สมัยก่อน โปรแกรมต้องเขียนและอ่านข้อมูลจาก RAM โดยตรงที่ตำแหน่ง Physical Address ซึ่งมีปัญหาอย่างมาก เช่น ข้อมูลกระจัดกระจาย (Fragmentation) และหน่วยความจำไม่พอ

OS สมัยใหม่แก้ปัญหานี้โดยการสร้าง **Virtual Address (ที่อยู่เสมือน)** และแบ่งหน่วยความจำออกเป็นบล็อกเล็กๆ ขนาดเท่าๆ กัน (ปกติบล็อกละ 4 KB) เรียกว่า **"Page"**

เมื่อโปรแกรมขอจองหน่วยความจำ:

1.  OS จะจัดสรรหน้ากระดาษเสมือน (Virtual Page) ให้
2.  OS จะใช้ตารางที่เรียกว่า Page Table ในการแมปว่า Virtual Page ที่ 1 ตรงกับ Physical Page ที่เท่าไหร่ใน RAM ของจริง
3.  ถ้า RAM ของจริงเต็ม OS จะคัดลอก Page ที่ไม่ได้ถูกใช้งานมานาน ถ่ายโอนลงไปเก็บไว้ในฮาร์ดดิสก์ (เรียกว่ากระบวนการ Swap out / Page out)
4.  และเมื่อโปรแกรมกลับมาเรียกใช้ข้อมูลนั้น OS ก็จะดึง Page จากฮาร์ดดิสก์กลับคืนมาใส่ใน RAM (Swap in / Page in)

**PagedAttention**
ในวงการ LLM Inference ระบบปฏิบัติการเสมือนนี้เป็นแรงบันดาลใจให้เกิดเทคโนโลยีที่ชื่อว่า **vLLM (PagedAttention)**

เนื่องจากบริบท (Context) หรือ Key-Value Cache ของโมเดลภาษามีความยาวที่ไม่แน่นอนและจองคิวบน VRAM ของการ์ดจอได้ยาก PagedAttention จึงประยุกต์ใช้แนวคิด Paging เพื่อหั่น KV Cache ออกเป็นบล็อกเล็กๆ แล้วเก็บกระจายๆ กันไปในช่องว่างของ VRAM ทำให้สามารถเสิร์ฟคำขอของยูสเซอร์ได้พร้อมกัน (Throughput) เพิ่มขึ้นถึง 300%
