# Implicit Gradient Descent (การเลื่อนลงตามเกรเดียนต์โดยนัย)

[← กลับสู่เนื้อหาบทเรียน](../lectures/project2_rag_prompt_engineering.md)

**Implicit Gradient Descent** เป็นสมมติฐานและการค้นพบทางทฤษฎีในวงการ AI เพื่อใช้อธิบายว่า ทำไมโมเดลภาษาขนาดใหญ่ (LLMs) จึงสามารถเรียนรู้จากตัวอย่างใน Prompt (In-Context Learning หรือ ICL) ได้ ทั้งๆ ที่ไม่ได้มีการอัปเดตน้ำหนักพารามิเตอร์ใดๆ เลย (ไม่มีการทำ Backpropagation)

ทฤษฎีนี้เสนอว่า กลไก Forward Pass ของ Self-Attention ใน Transformer แท้จริงแล้วสามารถเทียบเคียงทางคณิตศาสตร์ได้กับการทำ Gradient Descent ภายในโมเดลเอง (โดยนัย)

> 💡 **เปรียบเทียบให้เห็นภาพ (Analogy):** การเรียนรู้ปกติ (Explicit) เหมือนนักเรียนที่ต้องกลับบ้านไปจดและท่องจำสมุดโน้ต (อัปเดตน้ำหนัก) เพื่อให้เก่งขึ้น แต่ In-Context Learning (Implicit) เหมือนนักเรียนอัจฉริยะที่แค่มองเห็นตัวอย่างบนกระดาน 2-3 ข้อ ก็สามารถปรับความเข้าใจในสมองและแก้โจทย์ข้อต่อไปได้ทันทีโดยไม่ต้องไปจดแก้สมุดโน้ตเลย การเรียนรู้นี้เกิดขึ้นชั่วคราวในความจำระยะสั้นเท่านั้น

## คำอธิบายเชิงลึก (In-depth Explanation)

ปกติเมื่อเรา Fine-tune โมเดล (Explicit Gradient Descent) เราจะต้องอัปเดตค่าน้ำหนัก $W$ ให้กลายเป็น $W'$ เพื่อลด Loss
$$W' = W - \eta \nabla_W L$$

แต่ในการทำ **In-Context Learning (ICL)** งานวิจัย (เช่น ของ von Oswald et al., 2023) พบว่า การเอาตัวอย่าง Few-shot ป้อนเรียงต่อกันเข้าไปใน Prompt จะทำให้เกิดการประมวลผลเวกเตอร์อ้างอิงผ่าน $K$ (Key) และ $V$ (Value)

เมื่อคำถามใหม่ (Query: $Q$) เข้ามาเทียบกับเวกเตอร์เหล่านี้ กลไกสมการ Attention ของ Transformer:
$$\text{Attention}(Q, K, V) = \text{softmax}\left(Q K^T\right) V$$
จะมีพฤติกรรมทางคณิตศาสตร์ที่สร้างการประเมิน (Projection) คล้ายกับว่า มันกำลังคำนวณ **เมทริกซ์ปรับปรุงน้ำหนักเสมือน (Meta-Gradients)** จากชุดตัวอย่างนั้น และนำเมทริกซ์เสมือนนั้นไปคูณประยุกต์ใช้กับคำถามตัวใหม่ทันที

ดังนั้นการให้ตัวอย่างใน Prompt จึงเปรียบเสมือนการสั่งให้โมเดลรันกระบวนการ Gradient Descent ชั่วคราวลงไปบนตัวแปรสถานะความจำระยะสั้นของมันเอง (In-memory fine-tuning) ทำให้ผลลัพธ์ออกมาแม่นยำขึ้นโดยไม่ต้องแก้ไขฮาร์ดไดรฟ์ของโมเดลจริงๆ
