# PII (Personally Identifiable Information)

[← กลับสู่เนื้อหาบทเรียน](../lectures/project6_capstone_guide.md)

**PII (ข้อมูลที่สามารถระบุตัวตนได้)** คือข้อมูลสารสนเทศใดๆ ก็ตามที่สามารถนำมาใช้ ทั้งทางตรงและทางอ้อม เพื่อระบุตัวตน เจาะจงตัวบุคคล ติดต่อ หรือค้นหาตำแหน่งของบุคคลใดบุคคลหนึ่งได้

ในปัจจุบันที่มีกฎหมายคุ้มครองข้อมูลส่วนบุคคลที่เข้มงวด เช่น GDPR (ยุโรป) และ PDPA (ประเทศไทย) การจัดการข้อมูล PII ถือเป็นความท้าทายและความรับผิดชอบสูงสุดของวิศวกรข้อมูลและ AI Engineer

## คำอธิบายเชิงลึก (In-depth Explanation)
PII แบ่งออกเป็น 2 ประเภทหลักที่ต้องระวัง:

1.  **ข้อมูลระบุตัวตนโดยตรง (Direct Identifiers)**: ข้อมูลที่ใช้ชี้ตัวคนๆ นั้นได้ทันทีด้วยตัวมันเอง
    *   ชื่อ-นามสกุล
    *   เลขประจำตัวประชาชน / เลขพาสปอร์ต
    *   อีเมลส่วนตัว
    *   เบอร์โทรศัพท์
    *   ลายนิ้วมือ / ข้อมูลชีวมิติ (Biometrics)
2.  **ข้อมูลระบุตัวตนโดยอ้อม (Indirect Identifiers / Quasi-Identifiers)**: ข้อมูลที่ด้วยตัวมันเองอาจไม่รู้ว่าเป็นใคร แต่ถ้านำไปประกอบกับข้อมูลอื่น จะชี้เป้าได้ทันที
    *   รหัสไปรษณีย์
    *   เพศ, วันเกิด, เชื้อชาติ
    *   ข้อมูลประวัติการรักษาพยาบาล
    *   IP Address หรือ MAC Address ของคอมพิวเตอร์

## ตัวอย่างการนำไปใช้ (Practical Example)
**ปัญหา PII ใน Large Language Models (LLMs)**
หากเรานำเอกสารของบริษัท เช่น ประวัติลูกค้านับล้านคน ไปใช้เทรน (Fine-tune) โมเดลภาษาโดยไม่ลบ PII ออกก่อน โมเดลภาษาก็จะจดจำข้อมูลเหล่านั้นไว้ในค่าน้ำหนัก (Weights) 

เมื่อมีแฮ็กเกอร์หรือผู้ใช้ป้อน Prompt หลอกถามว่า "ข้อมูลบัตรเครดิตของคุณสมชายอยู่ที่ไหน?" โมเดลอาจจะ "หลอนความจริง" คายเลขบัตรเครดิตออกมาให้หน้าตาเฉย ซึ่งนอกจากจะผิดจริยธรรมแล้ว บริษัทยังอาจถูกฟ้องร้องระดับล้มละลายได้

**วิธีป้องกัน (Anonymization / Data Masking)**: 
AI Engineer มักจะใช้ไลบรารีอย่าง Presidio หรือสร้าง Regex กรองข้อมูล ก่อนจะส่งเข้ากระบวนการเทรน โดยแปลง PII เช่น `นายประหยัด โทร 081-xxx` ให้กลายเป็นตัวแปรสวมรอย เช่น `[PERSON_NAME] โทร [PHONE_NUMBER]` เพื่อไม่ให้โมเดลจำข้อมูลจริง
