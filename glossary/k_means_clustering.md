# K-Means Clustering (การจัดกลุ่มแบบเค-มีนส์)

[← กลับสู่เนื้อหาบทเรียน](../lectures/project2_rag_prompt_engineering.md)

**K-Means Clustering** เป็นอัลกอริทึมการเรียนรู้แบบไม่มีผู้สอน (Unsupervised Learning) ที่ได้รับความนิยมสูงสุดสำหรับการจัดกลุ่ม (Clustering) ข้อมูลดิบที่ไม่มีป้ายกำกับ (Unlabeled data)

เป้าหมายของมันคือการแบ่งชุดข้อมูลออกเป็นกลุ่มย่อยๆ จำนวน $K$ กลุ่ม โดยพยายามทำให้สมาชิกภายในกลุ่มเดียวกันมีความคล้ายคลึงกัน (ระยะทางใกล้กัน) มากที่สุด และทำให้แต่ละกลุ่มอยู่ห่างกันมากที่สุด

## คำอธิบายเชิงลึก (In-depth Explanation)

ในระบบ RAG ก้าวหน้าที่มีการทำ Index ค้นหาเวกเตอร์แบบ **IVF-PQ (Inverted File Index)** K-Means ถูกใช้เพื่อจัดกลุ่มเวกเตอร์เอกสารนับล้านๆ อันให้อยู่เป็นหมวดหมู่ เพื่อที่ตอนค้นหา จะได้ค้นเฉพาะกลุ่ม (Cluster) ที่เกี่ยวข้องเท่านั้น ไม่ต้องสแกนทั้งหมด

อัลกอริทึม K-Means มี 4 ขั้นตอนหลัก:

1.  **Initialization**: กำหนดค่า $K$ (จำนวนกลุ่ม) และสุ่มจุดศูนย์กลางกลุ่มเริ่มต้น $K$ จุด (เรียกว่า [Centroids](centroid.md)) ลงในปริภูมิ
2.  **Assignment**: ตรวจสอบเวกเตอร์ข้อมูลทุกตัว ว่าเวกเตอร์ไหนอยู่ใกล้ Centroid ตัวใดมากที่สุด (อิงตามระยะทาง Euclidean Distance) แล้วจับคู่เวกเตอร์นั้นเข้ากับกลุ่มนั้น
3.  **Update**: เมื่อได้กลุ่มแล้ว ให้คำนวณหาค่าเฉลี่ยตำแหน่งของสมาชิกทั้งหมดในกลุ่ม เพื่อกำหนดเป็นพิกัด "Centroid อันใหม่" แทนที่อันเดิม
4.  **Repeat**: ทำซ้ำข้อ 2 และ 3 ไปเรื่อยๆ จนกว่า Centroid จะหยุดเคลื่อนที่ (ข้อมูลเสถียรแล้ว)

สมการเป้าหมายคือการหาชุดพิกัด Centroid $\mu_i$ และเซตสมาชิก $S_i$ เพื่อให้ผลรวมกำลังสองของระยะทางภายในคลัสเตอร์ (WCSS) มีค่าน้อยที่สุด:
$$\arg \min_S \sum_{i=1}^K \sum_{x \in S_i} \| x - \mu_i \|^2$$

## ตัวอย่างการนำไปใช้ (Practical Example)

สมมติคุณมีเวกเตอร์ของบทความข่าว 1 แสนชิ้น คุณสั่งรัน K-Means ให้แบ่ง $K=100$ กลุ่ม โมเดลจะพยายามขยับตำแหน่ง 100 จุดไปเรื่อยๆ จนกระทั่งข่าวการเมือง ข่าวกีฬา ข่าวเศรษฐกิจ ฯลฯ ถูกรวมกองอยู่กับพิกัดศูนย์กลางกลุ่มใกล้เคียงกันโดยอัตโนมัติ
