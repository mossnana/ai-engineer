# JSON-RPC (เจซัน-อาร์พีซี)

[← กลับสู่เนื้อหาบทเรียน](../lectures/project3_agentic_workflows_tools.md)

**JSON-RPC (Remote Procedure Call)** เป็นโปรโตคอล (Protocol) รูปแบบหนึ่งสำหรับการให้คอมพิวเตอร์หรือโปรแกรมพูดคุยสั่งงานกันข้ามเครือข่าย โดยใช้รูปแบบข้อมูลเป็น JSON ซึ่งออกแบบมาให้มีความเรียบง่ายและมีน้ำหนักเบา (Lightweight) มากกว่าโปรโตคอลเก่าๆ อย่าง SOAP/XML-RPC

> 💡 **เปรียบเทียบให้เห็นภาพ (Analogy):** เหมือนการโทรสั่งอาหารดิลิเวอรีด้วยสคริปต์ที่เป๊ะมากๆ เช่น คุณโทรไปแล้วพูดรูปแบบเป๊ะๆ ว่า "Method: สั่งอาหาร, Params: ผัดไทย 1 จาน, ID: คิวที่ 1" ทางร้านก็จะตอบกลับมาในรูปแบบเป๊ะๆ เหมือนกันว่า "Result: รับทราบ กำลังทำ, ID: คิวที่ 1" ทำให้ทั้งสองฝั่ง (คอมพิวเตอร์) คุยกันรู้เรื่องโดยไม่มีความสับสน

## คำอธิบายเชิงลึก (In-depth Explanation)

คำว่า **RPC (Remote Procedure Call)** หมายถึงการที่โปรแกรมฝั่งไคลเอนต์ สั่งรันฟังก์ชันบางอย่างที่อยู่บนเครื่องเซิร์ฟเวอร์เสมือนกับว่าฟังก์ชันนั้นทำงานอยู่บนเครื่องไคลเอนต์เอง

ในมาตรฐาน **JSON-RPC (โดยเฉพาะเวอร์ชัน 2.0)** การสื่อสารจะส่งข้อมูลตายตัว 4 ส่วนเสมอ:

1.  **`jsonrpc`**: บ่งบอกเวอร์ชันของโปรโตคอล (ปัจจุบันมักเป็น "2.0")
2.  **`method`**: ชื่อของฟังก์ชันหรือเครื่องมือที่ต้องการจะเรียกใช้งาน (เป็น String)
3.  **`params`**: พารามิเตอร์ที่ต้องการส่งเข้าไปในฟังก์ชัน (เป็น Object หรือ Array ก็ได้)
4.  **`id`**: รหัสประจำตัวของคำสั่งนั้นๆ (ใช้สำหรับจับคู่ Request กับ Response ให้ตรงกันกรณีส่งพร้อมกันหลายคำสั่ง) _(เหมือนบัตรคิวสั่งอาหาร)_

ตัวอย่าง Request ของ JSON-RPC:

```json
{
  "jsonrpc": "2.0",
  "method": "subtract",
  "params": [42, 23],
  "id": 1
}
```

ตัวอย่าง Response:

```json
{
  "jsonrpc": "2.0",
  "result": 19,
  "id": 1
}
```

ในวงการ AI เรามักพบ JSON-RPC เป็นมาตรฐานพื้นฐานที่อยู่เบื้องหลัง **Model Context Protocol (MCP)** ซึ่งเป็นตัวกลางที่ทำให้อิดิเตอร์อย่าง Cursor หรือ Claude Desktop สามารถสั่งเรียกใช้งาน Tools ในเครื่องของผู้พัฒนาได้มาตรฐานเดียวกันหมด

## ตัวอย่างการนำไปใช้ (Practical Example)

การสื่อสารใน Web3 และ Blockchain: Node ของเครือข่าย Ethereum ใช้ JSON-RPC เป็นมาตรฐานหลักในการสื่อสารกับภายนอก หากแอปพลิเคชันของคุณต้องการดูยอดเงินในกระเป๋า หรือโอนเงิน มันจะสร้างก้อนข้อมูลรูปแบบ JSON-RPC ส่งผ่าน HTTP ไปยัง Node เซิร์ฟเวอร์
