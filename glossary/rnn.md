# Recurrent Neural Networks (RNN)

## คำจำกัดความ (Definition)

Recurrent Neural Networks (RNN) หรือ โครงข่ายประสาทเทียมแบบวนซ้ำ เป็นสถาปัตยกรรมของโครงข่ายประสาทเทียมที่ออกแบบมาสำหรับประมวลผลข้อมูลที่มีลักษณะเป็นลำดับ (Sequential Data) เช่น ข้อความ, เสียงพูด, หรือข้อมูลอนุกรมเวลา (Time-series)

จุดเด่นของ RNN คือการมี "สถานะซ่อนเร้น" (Hidden State) ที่ทำหน้าที่เสมือนหน่วยความจำ เก็บข้อมูลจากการคำนวณในอดีตมาใช้ร่วมกับข้อมูลในปัจจุบัน

> 💡 **เปรียบเทียบให้เห็นภาพ (Analogy):** ลองจินตนาการว่าคุณกำลังอ่านหนังสือนิยาย คุณไม่ได้เริ่มอ่านคำปัจจุบันด้วยสมองที่ว่างเปล่า แต่คุณจำเรื่องราวจากหน้าก่อนๆ (อดีต) มาทำความเข้าใจคำที่คุณกำลังอ่านอยู่ (ปัจจุบัน) "สถานะซ่อนเร้น" ก็คือความทรงจำเรื่องราวที่คุณเก็บสะสมมานั่นเอง

## แนวคิดหลัก (Key Concepts)

1. **การส่งผ่านสถานะซ่อนเร้น (Hidden State)**: การคำนวณที่เวลา $t$ จะประมวลผลข้อมูลนำเข้า $x_t$ ร่วมกับสถานะซ่อนเร้นจากเวลา $t-1$ ($h_{t-1}$) ทำให้โมเดลสามารถรักษาบริบทของลำดับข้อมูลที่ผ่านมาได้
   _(เหมือนการเอาความเข้าใจในประโยคที่แล้ว มารวมกับการอ่านประโยคนี้ เพื่อให้เข้าใจเรื่องราวต่อเนื่อง)_
2. **การแชร์น้ำหนัก (Parameter Sharing)**: ค่าน้ำหนัก (Weights) ของโมเดลถูกแชร์ใช้งานซ้ำในทุก ๆ time step (รอบเวลา) ช่วยลดจำนวนพารามิเตอร์เมื่อเทียบกับโครงข่ายประสาทเทียมแบบทั่วไป
   _(เหมือนคุณใช้ "กฎการอ่านภาษาไทย" ชุดเดียวกันเป๊ะๆ ไม่ว่าจะอ่านคำแรกของหนังสือ หรือคำที่ร้อยของหนังสือ)_
3. **ปัญหาเรื่องความจำระยะยาว (Long-Term Dependencies)**: RNN แบบดั้งเดิมมักจะเผชิญปัญหา Vanishing Gradient (เกรเดียนต์หาย) เมื่อลำดับข้อมูลยาวมากๆ ทำให้ความทรงจำในอดีตไกลๆ เลือนหายไป จึงทำให้ต้องมีการพัฒนา LSTM หรือ GRU เพื่อแก้ปัญหานี้
   _(เปรียบเหมือนถ้าคุณอ่านหนังสือรวดเดียว 500 หน้า คุณอาจจะลืมรายละเอียดเล็กๆ น้อยๆ ในหน้าแรกๆ ไปแล้ว)_

## ตัวอย่างโค้ดจำลองการทำงาน (Code Example)

เพื่อให้เห็นภาพการทำงานของการส่งผ่านสถานะซ่อนเร้น นี่คือตัวอย่างโค้ดจำลองการคำนวณ RNN แบบพื้นฐาน:

```python
import numpy as np

def rnn_forward(inputs, W_hx, W_hh, b_h):
    """
    inputs: ข้อมูลนำเข้าเป็นลำดับเวลา (sequence of inputs)
    W_hx: เมทริกซ์น้ำหนักสำหรับข้อมูลนำเข้า
    W_hh: เมทริกซ์น้ำหนักสำหรับสถานะซ่อนเร้นเดิม
    b_h: ไบแอส
    """
    # 1. กำหนดค่าเริ่มต้นของสถานะซ่อนเร้น h_0 เป็น 0 (เหมือนเริ่มอ่านหนังสือด้วยหัวโล่งๆ)
    h_t = np.zeros(W_hh.shape[0])
    hidden_states = []

    # 2. วนลูปตามลำดับเวลา (Sequential Processing) (อ่านไปทีละคำ)
    for x_t in inputs:
        # 3. คำนวณสถานะใหม่จากข้อมูลปัจจุบัน (x_t) และสถานะเก่า (h_t)
        h_t = np.tanh(np.dot(W_hx, x_t) + np.dot(W_hh, h_t) + b_h)
        hidden_states.append(h_t)

    return hidden_states
```

> [!TIP]
> สังเกตว่าในลูป `for x_t in inputs:` การคำนวณในรอบที่ `t` จะต้องพึ่งพาผลลัพธ์ `h_t` จากรอบที่ `t-1` เสมอ ทำให้โครงสร้างนี้มีความเป็นลำดับขั้น (Sequential) อย่างเลี่ยงไม่ได้ และไม่สามารถประมวลผลแบบรวบยอดพร้อมกันได้ _(เหมือนคุณต้องอ่านคำที่ 1, 2, 3 เรียงกันไป จะข้ามไปอ่านคำที่ 10 เลยโดยไม่รู้เนื้อเรื่องก่อนหน้าไม่ได้)_

## จุดเปลี่ยนสู่ยุค Transformer

ข้อจำกัดสำคัญของ RNN คือการทำงานต้องทำ **ตามลำดับเวลา** (Sequential Computation) ทำให้ไม่สามารถประมวลผลข้อมูลพร้อมกันในรูปแบบคู่ขนาน (Parallelization) ได้อย่างเต็มประสิทธิภาพบน GPU

สถาปัตยกรรม Transformer จึงเข้ามาแทนที่ด้วยกลไก Self-Attention ที่สามารถกวาดสายตาดูความสัมพันธ์ของคำทั้งประโยคได้พร้อมๆ กัน ทำให้การเทรนรวดเร็วและรองรับข้อมูลมหาศาลได้ดีกว่า

> 💡 **เปรียบเทียบให้เห็นภาพ (Analogy):** RNN เหมือนการต่อแถวซื้อของที่ทุกคนต้องเรียงคิวทำทีละคน (ประมวลผลทีละคำ) ส่วน Transformer เหมือนการมีพนักงานขายหลายคนช่วยกันให้บริการลูกค้าทุกคนพร้อมๆ กัน แล้วค่อยเอาข้อมูลมาเชื่อมโยงกัน (วิเคราะห์ทุกคำในประโยคพร้อมกัน) ทำให้ทำงานได้เร็วกว่ามาก!
